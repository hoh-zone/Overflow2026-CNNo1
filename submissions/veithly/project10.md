## Description / 项目简介

Memory Will turns agent handoff into a verifiable inheritance event. A reviewer kills a predecessor agent, chooses which memory scopes may survive, writes the scoped Memory Will to Walrus, and opens a cold successor that can only restore from the allowed Walrus-backed memory. The receipt shows the Walrus blob id, Sui object proof, redactions, replay trace, successor role, and restored output hash. Reviewer flow: open the workbench, click **Kill Agent**, choose a successor role, click **Write Memory Will**, wait for the Walrus pointer, then open the cold successor and inspect proof.

Memory Will 把 AI 代理交接做成可验证的“记忆遗嘱”。评审先杀死前任 agent，选择哪些记忆范围可以继承，系统把受限 Memory Will 写入 Walrus，再打开一个只能从允许范围恢复的冷启动 successor。收据会展示 Walrus blob id、Sui object proof、删减记录、replay trace、继任角色和恢复输出 hash。评审流程：打开 workbench，点击 **Kill Agent**，选择 successor role，点击 **Write Memory Will**，拿到 Walrus pointer 后打开冷启动 successor 并检查 proof。
